# Product and Category Copy Guide

## Product name

A good product name should be clear, searchable and factual.

Recommended structure:

```text
Product type + model/feature + key attribute
```

Do:

- keep brand/model if present;
- include size, material, color or compatibility when relevant;
- avoid keyword stuffing;
- avoid adding claims not present in the original data.

Do not:

- change SKU/article;
- invent certifications, guarantees, country of origin or technical specs;
- add “best”, “number one”, “official” unless provided by the user.

## Short description / brief

Use 1–2 compact sentences focused on the buyer’s decision:

- what the product is;
- who it is for;
- main benefit;
- key constraint or compatibility if important.

## Full description

Recommended structure:

1. Short intro.
2. Key benefits.
3. Specs or composition.
4. Usage scenarios.
5. Delivery/payment/return note only if verified and approved.

## HTML description

Use clean semantic markup:

```html
<p>...</p>
<ul>
  <li>...</li>
</ul>
```

Avoid inline styles unless the existing store uses them and the user approves.

## Category description

A useful category description should:

- explain what is inside the category;
- include common buyer intents;
- mention selection criteria;
- stay readable, not spammy;
- avoid repeating the same text across many categories.

Recommended length:

- short intro: 300–700 characters;
- expanded SEO text: only if the template has a dedicated lower text area or the user asks for it.

## Meta title

Recommended pattern:

```text
Category/Product name — buy/order at Store Name
```

Keep it concise and avoid duplicates.

## Meta description

Recommended pattern:

```text
Brief value proposition + product/category specifics + delivery/payment cue if verified.
```

Do not mention prices, delivery times or discounts unless the data is current and provided by the user/API.
